# Plan: NixOS Flake + Legacy default.nix

**Date:** 2026-05-12  
**Branch:** main  
**Topic:** nix-flake

---

## Goal

Make the `rtm` workspace installable and buildable on NixOS (and any system with Nix) by providing:

1. `flake.nix` — modern Nix flakes entry point exposing packages, a dev shell, and apps
2. `default.nix` — legacy compatibility shim (calls into the flake via `(builtins.getFlake …)` or
   re-implements the same derivations with `import <nixpkgs>`)

Both files must live at the repository root.

---

## Context & Constraints

### Crates in scope

| Crate | Binary | Native deps needed |
|---|---|---|
| `todotxt` | library only | none |
| `rtmcli` | `rtmcli` | none (pure Rust) |
| `rtmgui` | `rtmgui` | GTK3, X11/Wayland, OpenGL, libxkbcommon, wayland |
| `rtmapp/src-tauri` | `rtmapp` | GTK3, WebKitGTK 4.1, libsoup3, plus all of rtmgui's deps |

### Key constraints

- **Cargo workspace** at repo root; `Cargo.lock` is present (gitignored but present on disk — must
  be committed or the flake must use `importCargoLock` which reads it from the source tree).
  - **Action required:** Remove `Cargo.lock` from `.gitignore` (it is currently listed there).
    `buildRustPackage` / `importCargoLock` both need it committed.
- **Rust edition 2024** for `todotxt`, `rtmcli`, `rtmgui` — nixpkgs stable Rust may lag; use
  `nixpkgs-unstable` or pin a recent nixpkgs that ships Rust ≥ 1.85 (first stable with edition
  2024).
- **`rtmapp` (Tauri)** is the hardest to package: Tauri's build system (`cargo tauri build`) is
  not a plain `cargo build`; it also bundles a WebView frontend. The nixpkgs `tauriHook` (added
  ~2024) handles this, but it requires `wrapGAppsHook4` and `webkitgtk_4_1`. Scope this as a
  **best-effort / optional** package in the first iteration.
- **No async, no proc-macros** in `todotxt` or `rtmcli` — straightforward `buildRustPackage`.
- **`rtmgui`** links against GTK3 + X11/Wayland/OpenGL via `eframe`/`egui` + `rfd`. Needs
  `wrapGAppsHook` (GTK3 variant) and the same library set as `shell.nix`.
- The existing `shell.nix` uses `<nixpkgs>` (channel-pinned, impure). The flake will pin nixpkgs
  via `inputs` for reproducibility; `shell.nix` can be updated to delegate to the flake's
  `devShells.default`.

### Approach: `buildRustPackage` with `importCargoLock`

Use `pkgs.rustPlatform.buildRustPackage` with:
```nix
cargoLock.lockFile = ./Cargo.lock;
```
This is the simplest approach and avoids needing a pre-computed hash. It uses Import From
Derivation (IFD) which is fine for local builds.

## Steps

### 1. Commit `Cargo.lock`

- Remove `Cargo.lock` from `.gitignore` (line 125).
- Commit the lock file. This is mandatory for `importCargoLock` / `buildRustPackage`.
- **Acceptance:** `git ls-files Cargo.lock` shows the file tracked.

### 2. Create `flake.nix`

Location: `/flake.nix` (repo root).

#### 2a. Inputs

```nix
inputs = {
  nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
  # nixos-unstable ships Rust 1.86+ which supports edition 2024
  flake-utils.url = "github:numtide/flake-utils";
};
```

`flake-utils` provides `eachDefaultSystem` to avoid repeating per-system boilerplate.

#### 2b. Common native/build inputs per package

Define a helper `let` block inside `eachDefaultSystem`:

```nix
# Shared GUI deps (rtmgui)
guiDeps = with pkgs; [
  gtk3 glib gobject-introspection atk gdk-pixbuf pango cairo
  xorg.libxcb xorg.libX11 xorg.libXcursor xorg.libXrandr xorg.libXi
  libGL libxkbcommon wayland
];

# Additional deps for rtmapp (Tauri)
tauriDeps = guiDeps ++ (with pkgs; [
  webkitgtk_4_1 libsoup_3
]);
```

#### 2c. Package definitions

**`rtmcli`** (simplest):
```nix
rtmcli = pkgs.rustPlatform.buildRustPackage {
  pname = "rtmcli";
  version = "0.1.0";
  src = ./.;
  cargoLock.lockFile = ./Cargo.lock;
  cargoBuildFlags = [ "-p" "rtmcli" ];
  cargoTestFlags  = [ "-p" "rtmcli" "-p" "todotxt" ];
  meta = { ... };
};
```

**`rtmgui`**:
```nix
rtmgui = pkgs.rustPlatform.buildRustPackage {
  pname = "rtmgui";
  version = "0.1.0";
  src = ./.;
  cargoLock.lockFile = ./Cargo.lock;
  cargoBuildFlags = [ "-p" "rtmgui" ];
  nativeBuildInputs = [ pkgs.pkg-config pkgs.wrapGAppsHook ];
  buildInputs = guiDeps;
  meta = { ... };
};
```

**`rtmapp`** (Tauri — best-effort, may need iteration):
```nix
rtmapp = pkgs.rustPlatform.buildRustPackage {
  pname = "rtmapp";
  version = "0.1.0";
  src = ./.;
  cargoLock.lockFile = ./Cargo.lock;
  cargoBuildFlags = [ "-p" "rtmapp" ];
  nativeBuildInputs = [
    pkgs.pkg-config
    pkgs.wrapGAppsHook4   # GTK4 wrapper for Tauri v2
  ];
  buildInputs = tauriDeps;
  # Tauri needs WEBKIT_DISABLE_COMPOSITING_MODE on some systems
  WEBKIT_DISABLE_COMPOSITING_MODE = "1";
  meta = { ... };
};
```

> **Note:** Tauri v2 normally requires `cargo tauri build` (not plain `cargo build`) to bundle
> the frontend. Since the frontend is static files (`src/`), we can try plain `cargo build -p
> rtmapp` which builds only the Rust binary. The binary will look for frontend files at runtime
> relative to the executable. This may need a `postInstall` step to copy `rtmapp/src/` into the
> output. Alternatively, use `pkgs.tauriHook` if available in the pinned nixpkgs. **Flag this as
> a known risk** — see Risks section.

#### 2d. `devShells.default`

Replace the current `shell.nix` logic with a proper flake dev shell:
```nix
devShells.default = pkgs.mkShell {
  nativeBuildInputs = with pkgs; [
    rustc cargo rustfmt clippy rust-analyzer
    pkg-config
    wrapGAppsHook
    nodejs  # for rtmapp npm commands
  ];
  buildInputs = tauriDeps;
  shellHook = ''
    export LD_LIBRARY_PATH=/run/opengl-driver/lib/:${pkgs.lib.makeLibraryPath tauriDeps}
  '';
};
```

#### 2e. `apps` and `defaultPackage`

```nix
apps = {
  rtmcli = flake-utils.lib.mkApp { drv = packages.rtmcli; };
  rtmgui = flake-utils.lib.mkApp { drv = packages.rtmgui; };
};
defaultPackage = packages.rtmcli;  # or rtmgui
```

#### 2f. Full flake skeleton

```nix
{
  description = "Rusty Todo.txt Manager";

  inputs = {
    nixpkgs.url     = "github:NixOS/nixpkgs/nixos-unstable";
    flake-utils.url = "github:numtide/flake-utils";
  };

  outputs = { self, nixpkgs, flake-utils }:
    flake-utils.lib.eachDefaultSystem (system:
      let
        pkgs = nixpkgs.legacyPackages.${system};

        guiDeps = with pkgs; [
          gtk3 glib gobject-introspection atk gdk-pixbuf pango cairo
          xorg.libxcb xorg.libX11 xorg.libXcursor xorg.libXrandr xorg.libXi
          libGL libxkbcommon wayland
        ];
        tauriDeps = guiDeps ++ (with pkgs; [ webkitgtk_4_1 libsoup_3 ]);

        commonArgs = {
          src = ./.;
          cargoLock.lockFile = ./Cargo.lock;
          version = "0.1.0";
        };

        rtmcli = pkgs.rustPlatform.buildRustPackage (commonArgs // {
          pname = "rtmcli";
          cargoBuildFlags = [ "-p" "rtmcli" ];
          meta.description = "Rusty Todo.txt Manager — CLI";
        });

        rtmgui = pkgs.rustPlatform.buildRustPackage (commonArgs // {
          pname = "rtmgui";
          cargoBuildFlags = [ "-p" "rtmgui" ];
          nativeBuildInputs = [ pkgs.pkg-config pkgs.wrapGAppsHook ];
          buildInputs = guiDeps;
          meta.description = "Rusty Todo.txt Manager — egui GUI";
        });

        rtmapp = pkgs.rustPlatform.buildRustPackage (commonArgs // {
          pname = "rtmapp";
          cargoBuildFlags = [ "-p" "rtmapp" ];
          nativeBuildInputs = [ pkgs.pkg-config pkgs.wrapGAppsHook4 ];
          buildInputs = tauriDeps;
          WEBKIT_DISABLE_COMPOSITING_MODE = "1";
          meta.description = "Rusty Todo.txt Manager — Tauri app";
        });
      in {
        packages = { inherit rtmcli rtmgui rtmapp; };
        defaultPackage = rtmcli;

        apps = {
          rtmcli = flake-utils.lib.mkApp { drv = rtmcli; };
          rtmgui = flake-utils.lib.mkApp { drv = rtmgui; };
          rtmapp = flake-utils.lib.mkApp { drv = rtmapp; };
        };
        defaultApp = flake-utils.lib.mkApp { drv = rtmcli; };

        devShells.default = pkgs.mkShell {
          nativeBuildInputs = with pkgs; [
            rustc cargo rustfmt clippy rust-analyzer
            pkg-config wrapGAppsHook nodejs
          ];
          buildInputs = tauriDeps;
          shellHook = ''
            export LD_LIBRARY_PATH=/run/opengl-driver/lib/:${pkgs.lib.makeLibraryPath tauriDeps}
          '';
        };
      }
    );
}
```

### 3. Create `default.nix` (legacy shim)

Location: `/default.nix` (repo root).

Two viable approaches — choose the simpler one:

**Option A — delegate to flake (requires Nix ≥ 2.4 with flakes enabled):**
```nix
# default.nix — legacy entry point; delegates to flake.nix
(builtins.getFlake (toString ./.)).packages.${builtins.currentSystem}.rtmcli
```
This is minimal but requires flakes to be enabled in the user's Nix config.

**Option B — self-contained (works with classic `nix-build`):**
```nix
{ pkgs ? import <nixpkgs> {} }:
let
  guiDeps = with pkgs; [ gtk3 glib gobject-introspection atk gdk-pixbuf pango cairo
    xorg.libxcb xorg.libX11 xorg.libXcursor xorg.libXrandr xorg.libXi
    libGL libxkbcommon wayland ];
in
{
  rtmcli = pkgs.rustPlatform.buildRustPackage {
    pname = "rtmcli"; version = "0.1.0";
    src = ./.;
    cargoLock.lockFile = ./Cargo.lock;
    cargoBuildFlags = [ "-p" "rtmcli" ];
  };
  rtmgui = pkgs.rustPlatform.buildRustPackage {
    pname = "rtmgui"; version = "0.1.0";
    src = ./.;
    cargoLock.lockFile = ./Cargo.lock;
    cargoBuildFlags = [ "-p" "rtmgui" ];
    nativeBuildInputs = [ pkgs.pkg-config pkgs.wrapGAppsHook ];
    buildInputs = guiDeps;
  };
  # rtmapp omitted from default.nix for now (Tauri complexity)
}
```

**Recommendation:** Use Option B — it works without flakes enabled and is self-contained.
`nix-build -A rtmcli` and `nix-build -A rtmgui` will work out of the box.

### 4. Update `shell.nix` to delegate to flake devShell

Once the flake is in place, `shell.nix` can be simplified:
```nix
# shell.nix — legacy entry point for nix-shell / direnv use_nix
(import (fetchTarball "https://github.com/edolstra/flake-compat/archive/master.tar.gz") {
  src = ./.;
}).shellNix
```
Or keep the existing `shell.nix` as-is (it still works) and just add the flake alongside it.
The `.envrc` uses `use_nix` which reads `shell.nix` — if we want `use flake` instead, change
`.envrc` to `use flake`.

**Recommendation:** Keep `shell.nix` as-is for now; add a comment pointing to the flake. Update
`.envrc` to `use flake` if direnv ≥ 2.30 is available.

### 5. Handle `git+file://` source filtering

`buildRustPackage` copies the source via `src = ./.`. By default this includes everything in the
working tree. Add a `src` filter to avoid copying `target/`, `.git/`, `node_modules/`, etc.:

```nix
src = pkgs.lib.cleanSource ./.;
# or more targeted:
src = builtins.path {
  path = ./.;
  name = "rtm-source";
  filter = path: type:
    !(builtins.elem (baseNameOf path) [ "target" ".git" "node_modules" ".direnv" ]);
};
```

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| **Rust edition 2024 not in stable nixpkgs** | Medium | Pin `nixos-unstable`; Rust 1.85+ is in unstable as of early 2025 |
| **`rtmapp` Tauri binary can't find frontend files at runtime** | High | Add `postInstall` to copy `rtmapp/src/` into `$out/share/rtmapp/`; set `TAURI_WEBVIEW_ASSET_DIR` or patch `tauri.conf.json` to use an absolute path |
| **`rtmapp` build fails because `cargo build -p rtmapp` doesn't run Tauri's bundler** | Medium | The Rust binary itself compiles fine; only the bundling step is skipped. The binary should run if frontend assets are present. Use `pkgs.tauriHook` if available. |
| **`wrapGAppsHook` vs `wrapGAppsHook4`** | Low | `rtmgui` uses GTK3 → `wrapGAppsHook`; `rtmapp` uses WebKitGTK 4.1 → `wrapGAppsHook4`. Don't mix. |
| **`importCargoLock` fails for git dependencies** | Low | All deps are from crates.io; no git deps in `Cargo.lock`. Safe. |
| **`Cargo.lock` currently gitignored** | High | Must remove from `.gitignore` and commit before the flake can work in a clean checkout. |
| **`libsoup_3` package name** | Low | Verify nixpkgs attribute name: it may be `libsoup_3` or `libsoup3`. Check with `nix search nixpkgs libsoup`. |
| **`gobject-introspection` in `buildInputs` vs `nativeBuildInputs`** | Low | `gobject-introspection` should be in `nativeBuildInputs` (it's a build-time tool). Move it there. |

## Open Questions

1. **Should `rtmapp` be included in the initial flake?** It's the most complex to package. Could
   ship `rtmcli` + `rtmgui` first and add `rtmapp` in a follow-up.

2. **`flake-utils` vs manual `forAllSystems`?** `flake-utils` is a common dependency but adds an
   input. Alternative: write `eachDefaultSystem` manually (5 lines). Preference?

3. **`defaultPackage` — `rtmcli` or `rtmgui`?** `rtmcli` is simpler and more universally
   buildable (no GUI deps). Recommend `rtmcli` as default.

4. **`use flake` in `.envrc`?** Requires direnv ≥ 2.30 and `nix-direnv`. If Master uses
   `nix-direnv`, switching `.envrc` from `use_nix` to `use flake` gives a faster, cached dev
   shell. Otherwise keep `use_nix`.

5. **NixOS module?** Out of scope for now, but if Master wants to install `rtmcli` system-wide
   via `environment.systemPackages`, the flake's `packages` output is sufficient — no NixOS
   module needed.
